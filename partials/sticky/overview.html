Overview

Sticky States

A Sticky State is state with a special marker. The marked state is the root of a state tree which gets a different lifecycle than normal states. The DOM and $scope of a Sticky State (and all activated substates) is retained until the parent state of the sticky state is exited. The sticky state tree will remain in an "inactivated" status when transitioning to a direct sibling state, or to a child of a sibling state.

This feature is based on a "tabs" use case. The use case requires that a tab in the application does not get exited when switching to one of the other tabs. The user should be able to switch back and forth between tabs without any interruption in their workflow. The URL should reflect only the state tree of the active tab.

For the tabs use case, Sticky States work best when also marked as Deep State Redirect
